# Util Methods

本章节介绍 Lodash 的工具类方法（Util），这些方法多用于函数组合、流程控制、数据生成与辅助判断，是构建复杂逻辑时的重要基础工具。

## attempt

`_.attempt(func, [args])`

尝试调用 `func`，返回结果或捕获的错误对象。适用于避免因异常中断流程的场景。

#### Since
3.0.0

#### Arguments
1. `func` *(Function)*: 要尝试调用的函数。
2. `[args]` *(...&#42;)*: 传递给 `func` 的参数。

#### Returns
*(&#42;)*: 返回 `func` 的执行结果，若抛出异常则返回错误对象。

#### Example
```js
var attempts = _.attempt(function(divider) {
  return 1 / divider;
}, 0);

console.log(attempts instanceof Error);
// => true

var attempts = _.attempt(function(divider) {
  return 1 / divider;
}, 2);

console.log(attempts);
// => 0.5
```

## bindAll

`_.bindAll(object, methodNames)`

将 `object` 上指定的方法绑定到对象本身，确保方法调用时 `this` 始终指向 `object`。常用于避免方法丢失上下文。

#### Since
0.1.0

#### Arguments
1. `object` *(Object)*: 要绑定的对象。
2. `methodNames` *(...(string|string&#91;&#93;))*: 要绑定的方法名列表。

#### Returns
*(Object)*: 返回 `object`。

#### Example
```js
var view = {
  'label': 'docs',
  'onClick': function() {
    console.log('clicked ' + this.label);
  }
};

_.bindAll(view, ['onClick']);
view.onClick();
// => 'clicked docs'
```

## cond

`_.cond(pairs)`

创建一个函数，该函数遍历 `pairs` 中的条件-处理函数对，返回第一个谓词返回真值对应的处理函数结果。

#### Since
4.0.0

#### Arguments
1. `pairs` *(Array)*: 条件-处理函数对数组，每个元素为 `[predicate, handler]`。

#### Returns
*(Function)*: 返回新的条件处理函数。

#### Example
```js
var func = _.cond([
  [_.matches({ 'a': 1 }),   function(o) { return 'matches A'; }],
  [_.conforms({ 'b': function(n) { return n > 1; }), function(o) { return 'matches B'; }],
  [_.stubTrue, function() { return 'default'; }]
]);

func({ 'a': 1, 'b': 2 });
// => 'matches A'

func({ 'a': 0, 'b': 2 });
// => 'matches B'

func({ 'a': 0, 'b': 0 });
// => 'default'
```

## conforms

`_.conforms(source)`

创建一个函数，该函数对传入的对象调用 `source` 中每个属性对应的谓词，全部通过则返回 `true`。

#### Since
4.0.0

#### Arguments
1. `source` *(Object)*: 含属性谓词的对象。

#### Returns
*(Function)*: 返回新的谓词函数。

#### Example
```js
var isUser = _.conforms({
  'age': function(age) { return age >= 18; },
  'active': function(active) { return active === true; }
});

isUser({ 'age': 20, 'active': true });
// => true

isUser({ 'age': 16, 'active': true });
// => false
```

## constant

`_.constant(value)`

创建一个返回固定 `value` 的函数。

#### Since
0.1.0

#### Arguments
1. `value` *(&#42;)*: 要返回的固定值。

#### Returns
*(Function)*: 返回新的常量函数。

#### Example
```js
var objects = _.times(2, _.constant({ 'a': 1 }));

console.log(objects);
// => [{ 'a': 1 }, { 'a': 1 }]

console.log(objects[0] === objects[1]);
// => true
```

## flow

`_.flow([funcs])`

创建一个函数，该函数从左至右执行传入的函数序列，将前一个函数的返回值作为下一个函数的入参。

#### Since
3.0.0

#### Arguments
1. `[funcs]` *(...(Function|Function&#91;&#93;))*: 要组合的函数序列。

#### Returns
*(Function)*: 返回新的组合函数。

#### Example
```js
function square(n) {
  return n * n;
}

var addSquare = _.flow([_.add, square]);
addSquare(1, 2);
// => 9
```

## flowRight

`_.flowRight([funcs])`

创建一个函数，该函数从右至左执行传入的函数序列，将前一个函数的返回值作为下一个函数的入参。

#### Since
3.0.0

#### Arguments
1. `[funcs]` *(...(Function|Function&#91;&#93;))*: 要组合的函数序列。

#### Returns
*(Function)*: 返回新的组合函数。

#### Example
```js
function square(n) {
  return n * n;
}

var squareAdd = _.flowRight([_.add, square]);
squareAdd(1, 2);
// => 5
```

## identity

`_.identity(value)`

返回传入的第一个参数，常用于函数式编程中的默认处理逻辑。

#### Since
0.1.0

#### Arguments
1. `value` *(&#42;)*: 要返回的值。

#### Returns
*(&#42;)*: 返回 `value`。

#### Example
```js
var objects = [{ 'a': 1 }, { 'a': 2 }];

_.map(objects, _.identity);
// => [{ 'a': 1 }, { 'a': 2 }]
```

## iteratee

`_.iteratee([func=_.identity])`

将一个值转换为对应的迭代器函数，支持函数、对象、属性名、属性路径等多种简写形式。

#### Since
0.1.0

#### Arguments
1. `[func=_.identity]` *(&#42;)*: 要转换为迭代器的值。

#### Returns
*(Function)*: 返回对应的迭代器函数。

#### Example
```js
var users = [
  { 'user': 'barney', 'age': 36 },
  { 'user': 'fred',   'age': 40 }
];

_.filter(users, _.iteratee({ 'user': 'barney' }));
// => [{ 'user': 'barney', 'age': 36 }]

_.map(users, _.iteratee('user'));
// => ['barney', 'fred']
```

## method

`_.method(path, [args])`

创建一个函数，该函数调用给定对象上 `path` 对应的方法，并传入指定参数。

#### Since
3.0.0

#### Arguments
1. `path` *(Array|string)*: 要调用的方法路径。
2. `[args]` *(...&#42;)*: 调用方法时传入的参数。

#### Returns
*(Function)*: 返回新的调用函数。

#### Example
```js
var objects = [{ 'a': { 'b': _.constant(1) } }, { 'a': { 'b': _.constant(2) } }];

_.map(objects, _.method('a.b'));
// => [1, 2]
```

## methodOf

`_.methodOf(object, [args])`

创建一个函数，该函数调用固定 `object` 上指定路径对应的方法，并传入参数。

#### Since
3.0.0

#### Arguments
1. `object` *(Object)*: 要调用方法的对象。
2. `[args]` *(...&#42;)*: 调用方法时传入的参数。

#### Returns
*(Function)*: 返回新的调用函数。

#### Example
```js
var object = { 'a': _.constant(1), 'b': _.constant(2) };

_.map(['a', 'b'], _.methodOf(object));
// => [1, 2]
```

## mixin

`_.mixin([object=lodash], source, [options={}])`

将 `source` 上的方法混入到 `object` 中，常用于扩展 Lodash 本身的功能。

#### Since
0.1.0

#### Arguments
1. `[object=lodash]` *(Function)*: 要混入方法的对象，默认是 Lodash 本身。
2. `source` *(Object)*: 要混入的方法集合。
3. `[options={}]` *(Object)*: 配置项。
4. `[options.chain=true]` *(boolean)*: 是否开启链式调用。

#### Returns
*(Function)*: 返回 `object`。

#### Example
```js
function greet(greeting, name) {
  return greeting + ' ' + name;
}

_.mixin({ 'greet': greet });
_.greet('hi', 'fred');
// => 'hi fred'

_('fred').greet('hi').value();
// => 'hi fred'
```

## noop

`_.noop()`

一个空函数，不执行任何操作，常用于占位或默认回调。

#### Since
0.1.0

#### Returns
*(undefined)*: 返回 `undefined`。

#### Example
```js
_.times(2, _.noop);
// => [undefined, undefined]
```

## nthArg

`_.nthArg([n=0])`

创建一个函数，该函数返回第 `n` 个传入的参数，`n` 为负数时从末尾计数。

#### Since
4.0.0

#### Arguments
1. `[n=0]` *(number)*: 要返回的参数位置。

#### Returns
*(Function)*: 返回新的参数提取函数。

#### Example
```js
var func = _.nthArg(1);
func(1, 2, 3);
// => 2

var func = _.nthArg(-1);
func(1, 2, 3);
// => 3
```

## over

`_.over([iteratees=[_.identity]])`

创建一个函数，该函数对传入的值分别调用多个迭代器，返回所有结果组成的数组。

#### Since
4.0.0

#### Arguments
1. `[iteratees=[_.identity]]` *(...(Function|Function&#91;&#93;))*: 要执行的迭代器集合。

#### Returns
*(Function)*: 返回新的分发函数。

#### Example
```js
var func = _.over([Math.max, Math.min]);
func(1, 2, 3, 4);
// => [4, 1]
```

## overEvery

`_.overEvery([predicates=[_.identity]])`

创建一个函数，该函数对传入的值调用所有谓词，全部返回真值时才返回 `true`。

#### Since
4.0.0

#### Arguments
1. `[predicates=[_.identity]]` *(...(Function|Function&#91;&#93;))*: 要执行的谓词集合。

#### Returns
*(Function)*: 返回新的检查函数。

#### Example
```js
var func = _.overEvery([Boolean, _.isFinite]);
func(1);
// => true

func(0);
// => true

func(null);
// => false
```

## overSome

`_.overSome([predicates=[_.identity]])`

创建一个函数，该函数对传入的值调用所有谓词，任意一个返回真值就返回 `true`。

#### Since
4.0.0

#### Arguments
1. `[predicates=[_.identity]]` *(...(Function|Function&#91;&#93;))*: 要执行的谓词集合。

#### Returns
*(Function)*: 返回新的检查函数。

#### Example
```js
var func = _.overSome([_.isNull, _.isUndefined]);
func(null);
// => true

func(undefined);
// => true

func(1);
// => false
```

创建一个函数，该函数返回传入对象上 `path` 对应的属性值。

#### Since
0.1.0

#### Arguments
1. `path` *(Array|string)*: 要获取的属性路径。

#### Returns
*(Function)*: 返回新的属性获取函数。

#### Example
```js
var objects = [
  { 'a': { 'b': 2 } },
  { 'a': { 'b': 1 } }

_.map(objects, _.property('a.b'));
// => [2, 1]
```

## propertyOf

`_.propertyOf(object)`

创建一个函数，该函数返回固定 `object` 上指定路径对应的属性值。

#### Since
3.0.0

#### Arguments
1. `object` *(Object)*: 要读取属性的对象。

#### Returns
*(Function)*: 返回新的属性获取函数。

#### Example
```js
var object = { 'a': 1, 'b': 2 };

var get = _.propertyOf(object);

// => 1

get(['b']);
// => 2
```

## range

`_.range([start=0], end, [step=1])`

生成一个从 `start` 到 `end`（不包含）的等差数列数组，步长为 `step`。

#### Since
0.1.0

#### Arguments
1. `[start=0]` *(number)*: 起始值。
2. `end` *(number)*: 结束值（不包含）。
3. `[step=1]` *(number)*: 步长。

#### Returns
*(Array)*: 返回生成的数组。

#### Example
```js
_.range(4);
// => [0, 1, 2, 3]

_.range(1, 5);
// => [1, 2, 3, 4]

_.range(0, 20, 5);
// => [0, 5, 10, 15]

_.range(5, 1);
// => []

_.range(0, -4, -1);
// => [0, -1, -2, -3]
```

## rangeRight

`_.rangeRight([start=0], end, [step=1])`

同 `_.range`，但生成的数组是从 `end` 到 `start` 逆序排列的。

#### Since
4.0.0

#### Arguments
1. `[start=0]` *(number)*: 起始值。
2. `end` *(number)*: 结束值（不包含）。
3. `[step=1]` *(number)*: 步长。

#### Returns
*(Array)*: 返回生成的逆序数组。

#### Example
```js
_.rangeRight(4);
// => [3, 2, 1, 0]

_.rangeRight(1, 5);
// => [4, 3, 2, 1]

_.rangeRight(0, 20, 5);
// => [15, 10, 5, 0]

_.rangeRight(5, 1);
// => []

_.rangeRight(0, -4, -1);
// => [-3, -2, -1, 0]
```

## runInContext

`_.runInContext([context=root])`

创建一个使用指定上下文的新 Lodash 实例，常用于避免污染全局环境。

#### Since
1.1.0

#### Arguments
1. `[context=root]` *(Object)*: 要使用的上下文对象。

#### Returns
*(Function)*: 返回新的 Lodash 函数。

#### Example
```js
var lodash = _.runInContext();

lodash === _;
// => false

lodash.mixin({ 'add': function(a, b) { return a + b; } });
lodash.add(1, 2);
// => 3
```

## stubFalse

`_.stubFalse()`

一个始终返回 `false` 的函数，常用于需要固定返回假值的场景。

#### Since
4.0.0

#### Returns
*(boolean)*: 返回 `false`。

#### Example
```js
_.times(2, _.stubFalse);
// => [false, false]
```

## stubTrue

`_.stubTrue()`

一个始终返回 `true` 的函数，常用于需要固定返回真值的场景。

#### Since
4.0.0

#### Returns
*(boolean)*: 返回 `true`。

#### Example
```js
_.times(2, _.stubTrue);
// => [true, true]
```

## times

`_.times(n, [iteratee=_.identity])`

调用 `iteratee` 共 `n` 次，返回每次调用结果组成的数组。

#### Since
0.1.0

#### Arguments
1. `n` *(number)*: 调用次数。
2. `[iteratee=_.identity]` *(Function)*: 每次调用的函数。

#### Returns
*(Array)*: 返回结果数组。

#### Example
```js
_.times(3, function(n) {
  return n * 2;
});
// => [0, 2, 4]
```

## toPath

`_.toPath(value)`

将一个值转换为属性路径数组，支持字符串路径、数组路径等形式的转换。

#### Since
4.0.0

#### Arguments
1. `value` *(&#42;)*: 要转换的值。

#### Returns
*(Array)*: 返回属性路径数组。

#### Example
```js
_.toPath('a.b.c');
// => ['a', 'b', 'c']

_.toPath(['a', 'b', 'c']);
// => ['a', 'b', 'c']
```

## uniqueId

`_.uniqueId([prefix=''])`

生成一个全局唯一的 ID，可选添加前缀。

#### Since
0.1.0

#### Arguments
1. `[prefix='']` *(string)*: ID 的前缀。

#### Returns
*(string)*: 返回唯一 ID。

#### Example
```js
_.uniqueId('id_');
// => 'id_1'

_.uniqueId('id_');
// => 'id_2'
```
