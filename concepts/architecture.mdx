## 什么是 Lodash 架构

Lodash 是一个以<Mark bold>模块化</Mark>、<Mark bold>一致性</Mark>、<Mark bold>高性能</Mark>为核心目标的 JavaScript 工具库。在架构层面，它采用统一的源码组织方式、标准的模块导出格式以及清晰的功能分类策略，使开发者既可以在浏览器中直接使用完整构建，也能在 Node.js 或现代构建工具中按需引入单个方法，获得最优的包体积与运行性能。

## 模块导出方式：UMD

Lodash 默认以 <Link href="https://github.com/umdjs/umd">UMD（Universal Module Definition）</Link> 格式导出，这意味着同一份构建产物可以同时适配多种模块环境：

这种 UMD 设计保证了 Lodash 在从传统脚本到现代模块系统的整个 JavaScript 生态中都能无缝使用。

## 核心设计理念

### 一致性

Lodash 所有方法都遵循统一的 API 设计风格：参数顺序、返回值类型、错误处理方式均保持一致。这降低了学习成本，使开发者在熟悉少量方法后，可以快速上手其他方法。

### 稳定性优先

Lodash 目前已经进入 <Mark bold>Feature-Complete</Mark> 成熟阶段，不再激进地新增功能，而是将重点放在稳定性、安全性与长期可维护性上。更多治理相关信息可参考 <Link href="/project/governance">Governance</Link> 与 <Link href="/project/contributing">Contributing</Link>。

### 渐进增强与按需使用

通过分类模块、单方法文件和多种构建格式，Lodash 支持从“全量引入”到“单方法引入”的渐进使用方式，既能满足快速原型开发，也能满足生产环境对包体积和性能的严格要求。
